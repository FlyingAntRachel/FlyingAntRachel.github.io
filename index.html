荣品king rk3399开发板刷入unbuntu系统后使用vnc远程控制

发表于 2023-05-23 分类于 unbuntu

前言：最近小黄鱼淘了个荣品king-rk3399arm开发板，打算部署docker-android。因为hdmi接口连接显示屏不能显示，所有想通过vnc远程连接控制桌面，记录下遇到的问题。（unbuntu系统为20.01，内核5.1，镜像是开发板对应的镜像自带图形界面，虽然系统里已经安装了xfce桌面环境但是连接vnc后画面全灰）

1. 查看，安装，设置桌面环境为xfce

查看当前桌面环境：

echo $XDG_CURRENT_DESKTOP

如果未安装桌面，则返回空,这时候可以在命令行中运行以下命令来查看已安装的桌面环境：

ls /usr/share/xsessions/

命令的输出将列出所有已安装的桌面环境。例如，你可能会看到以下输出：

xfce.desktop

没有安装可以执行，如果vnc连接后显示不正常建议重新安装下面桌面环境：

sudo apt-get install xserver-common
sudo apt-get install xwayland
sudo apt-get install gnome-session
sudo apt-get install gnome-core xfce4

在命令行中运行以下命令来设置桌面环境变量：

export XDG_CURRENT_DESKTOP=xfce.desktop

2. 安装VNC

sudo apt-get install tightvncserver

启动vnc设置密码生成配置文件

vncserver

使用文本编辑软件nano编辑内容，如果没有安装：sudo apt install nano

sudo nano .vnc/xstartup

编辑替换下面内容之后，依次按下 ctrl+x >> y >> enter,保存内容：

#!/bin/sh
unset SESSION_MANAGER
unset DBUS_SESSION_BUS_ADDRESS
startxfce4 &
[ -x /etc/vnc/xstartup ] && exec /etc/vnc/xstartup
[ -r $HOME/.Xresources ] && xrdb $HOME/.Xresources
xsetroot -solid grey
vncconfig -iconic &

设置权限，否则你可能最后一直都是灰色，鼠标为叉：

sudo chmod +x ~/.vnc/xstartup

重启启动vnc,这里的 :1 代表连接端口是 ip:5901, :2 >> ip:5902,依次类推。

vncserver -kill :1
vncserver -geometry 1920x1080 :1

window下载vnc连接控制端软件：VNC View,下载地址：https://www.realvnc.com/en/connect/download/combined/

下载安装启动后，在输入框输入 你的ip:5901 就可以连接控制桌面了：
